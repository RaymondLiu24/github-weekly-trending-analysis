## 個人精選

### [HKUDS/RAG-Anything](https://github.com/HKUDS/RAG-Anything) 🧠（本週排名：13）

- **它在做什麼：** All-in-One RAG Framework，整合多模態文件解析（PDF、Word、Excel、圖片）、知識圖譜構建與語意檢索，一個框架涵蓋 RAG pipeline 全程，無需自行拼接各環節工具。
- **跟我的方向有什麼關係：** Personal AI super project 的核心挑戰之一是把異構資料（日記、履歷、掃描文件）統一餵進檢索系統；RAG-Anything 的多模態解析層可直接充當前處理管道，節省自組 parser + chunker 的工程量，讓重心回到個人化邏輯。
- **值得注意的技術選擇：** 採用 knowledge graph + vector 混合索引，可同時做「語意相似搜尋」和「關係推理」兩種查詢模式，比純 vector RAG 更適合長期累積、帶有實體關係的個人資料庫（例如「哪些工作經歷跟這個技能節點有關」）。

### [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent) 🧠（本週排名：16）

- **它在做什麼：** 「越用越聰明」的 AI agent——內建學習迴圈，從每次 session 自動提煉技能、改善 prompt、搜尋過去對話，並跨 session 建立使用者的個人模型。v0.11.0（4/23 發布）將 CLI 完整重寫為 React/Ink 互動介面，累計 119K 星。
- **跟我的方向有什麼關係：** 這是目前最接近 super project 理想形態的現成框架——其「從使用經驗自動創建技能」的機制，正是你想用日記與工作記錄建立個人記憶層的核心邏輯。可作為設計參考，或直接 fork 作為個人 agent 的 backbone。
- **值得注意的技術選擇：** 模型完全中立（相容 OpenRouter 200+ 模型及任意本地 endpoint）；技能以純 markdown 描述而非程式碼，讓定義 agent 行為不需工程背景，也讓個人知識的版本管理變得自然。

### [yoloshii/ClawMem](https://github.com/yoloshii/ClawMem) 🧠（本週排名：20）

- **它在做什麼：** 本地部署的 AI agent 記憶中介層，把 markdown 筆記、專案文件、研究資料轉成持久化記憶，採用 hybrid RAG（向量 + BM25 + 意圖分類）在每次 prompt 自動召回相關上下文。原生支援 Claude Code、Hermes Agent、OpenClaw。
- **跟我的方向有什麼關係：** 這是 super project 的「記憶 middleware」最具體候選——它不取代 agent，只是在 agent 與知識庫之間加一層智慧召回；把個人日記、履歷餵進去後，所有 coding agent session 都能存取個人脈絡，無需每次手動貼上背景資料。
- **值得注意的技術選擇：** 「self-evolving memory notes」機制——系統根據使用頻率自動調整記憶權重，最常召回的脈絡會被精煉為更短摘要，長期使用下 token 成本自動下降，記憶品質反而上升。

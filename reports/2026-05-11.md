## 個人精選

### [rohitg00/agentmemory](https://github.com/rohitg00/agentmemory) 🧠（本週排名：9）

- **它在做什麼：** 為 AI coding agent 提供持久化記憶系統，讓 agent 能夠跨 session 保留上下文、學習模式與偏好，而非每次從零開始。
- **跟我的方向有什麼關係：** 直接對應個人 AI 助理 super project 中最核心的設計問題——記憶系統架構。本週它與 rowboat（有記憶的 AI coworker）同步入榜，顯示記憶層已被社群視為 agent 的必要基礎建設，而非附加功能。
- **值得注意的技術選擇：** 定位在「coding agent」場景（而非通用 AI 助理），聚焦於程式碼上下文的記憶與檢索，可作為更廣泛個人記憶系統的技術參考起點。

### [rowboatlabs/rowboat](https://github.com/rowboatlabs/rowboat) 🧠（本週排名：21）

- **它在做什麼：** 開源的 AI 工作夥伴（coworker），核心特色是帶有持久記憶的工作協作，可跨 session 保留上下文與工作模式。
- **跟我的方向有什麼關係：** 與 agentmemory（#9）同週入榜，構成「agent 記憶周」的雙重信號。rowboat 的「coworker」定位——強調協作而非指令執行——提供了個人 AI 助理產品定位的重要對照：記憶系統如何影響使用者與 AI 的關係模式。
- **值得注意的技術選擇：** 記憶系統與任務規劃層採解耦設計，有助於獨立替換記憶後端；以開源形式釋出完整 agent 協作架構，可直接參考其 memory schema 設計。

### [VectifyAI/PageIndex](https://github.com/VectifyAI/PageIndex) 📚（本週排名：24）

- **它在做什麼：** 面向 reasoning 模型的文件索引系統，從長文件提取結構化頁面索引，支援推理式檢索而非單純向量相似度搜尋。
- **跟我的方向有什麼關係：** 直接對應 PKM / 第二大腦場景：當知識庫文件量大時，傳統 RAG 的語意搜尋無法處理需要多步推理的查詢，PageIndex 可顯著提升個人知識庫的深度問答能力。
- **值得注意的技術選擇：** 不依賴傳統向量嵌入，改用 LLM 生成頁面摘要索引，使 reasoning 模型（如 o3、claude-opus）能直接在索引層進行推理——有別於現行主流 RAG pipeline，是值得深入研究的架構差異點。

### [LearningCircuit/local-deep-research](https://github.com/LearningCircuit/local-deep-research) 📚（本週排名：15）

- **它在做什麼：** 完全本地執行的深度研究系統，在 SimpleQA 基準達到 95% 準確率，不依賴任何雲端服務。
- **跟我的方向有什麼關係：** 對個人知識庫工具具有直接參考價值——把個人資料（日記、筆記、文件）與公開知識整合時，本地執行的深度研究系統能兼顧隱私與查詢能力，是第二大腦架構理想的搜尋後端選項。
- **值得注意的技術選擇：** 強調完全本地執行且達到高 benchmark 準確率，暗示其架構做了針對本地推理資源的深度優化（可能結合分層索引與 retrieval-augmented generation 的混合策略），隱私優先設計具有差異化競爭力。
